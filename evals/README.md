# Hairstyle Recommendation Evaluation Harness

Local prompt evaluation using Ollama and LangChain.

## Setup

### Prerequisites
- `uv` >=0.9.0 (Python package manager)
- `ollama` running locally on port 11434
- Python 3.11+

### Installation

```bash
cd evals
uv sync
```

### Export Production Prompts

From repo root:

```bash
pnpm eval:export-prompts
```

This generates `evals/hairstyle/prompts.generated.json` from production code:
- Analyze prompts (ko + en)
- Recommend prompts for 3 representative inputs
- Candidate hairstyle lists
- JSON schemas

## Usage

### Basic evaluation (all stages, all models, all locales)

```bash
uv run python hairstyle/run.py
```

### Analyze stage only (vision models)

```bash
uv run python hairstyle/run.py --stage analyze
```

### Recommend stage only (text models)

```bash
uv run python hairstyle/run.py --stage recommend
```

### Specific models

```bash
uv run python hairstyle/run.py --models qwen3-vl:8b,qwen:14b
```

### Korean only

```bash
uv run python hairstyle/run.py --locale ko
```

### Limit fixtures per model

```bash
uv run python hairstyle/run.py --limit 2
```

### Full example

```bash
uv run python hairstyle/run.py \
  --stage all \
  --models all \
  --locale both \
  --limit 3
```

## Metrics

### Analyze Stage (Vision)
- **json_valid_rate**: % of responses successfully parsed as JSON
- **schema_valid_rate**: % of JSON matching FaceAnalysis schema
- **gender_accuracy**: % of detected genders matching ground truth
- **face_shape_top1**: % of detected face shapes matching ground truth
- **face_shape_top1_confident**: accuracy on fixtures where we're confident in the label
- **latency_p50/p95**: response time percentiles

### Recommend Stage (Text)
- **json_valid_rate**: % of responses parsed as JSON
- **schema_valid_rate**: % of JSON matching ProviderRecommendation array
- **candidate_id_adherence**: % of recommendations using only candidate IDs
- **locale_compliance_rate**: % with Hangul for Korean locale
- **latency_p50/p95**: response time percentiles

### Adding Fixtures

1. Add image to `docs/services/beauty/hairstyle-recommendation/data/`
2. Add entry to `hairstyle/fixtures.json`:
   ```json
   {
     "imagePath": "../../../docs/services/beauty/...",
     "gender": "male|female",
     "faceShape": "oval|round|square|heart|oblong|diamond|triangle",
     "faceShapeConfident": true|false,
     "name": "identifier"
   }
   ```

Generated to `hairstyle/reports/report-YYYYMMDD-HHmmss.md`.

Includes:
- Summary table (all models × stages × locales)
- Detailed metrics per model
- Sample failures
- Timestamp

## Development

### Project Structure

```
evals/
├── pyproject.toml           # uv project config
├── hairstyle/
│   ├── run.py              # Typer CLI (main)
│   ├── schemas.py          # Pydantic models
│   ├── fixtures.json       # Test photos + ground truth
│   ├── prompts.generated.json (generated by export-prompts)
│   └── reports/            # Report output directory
└── README.md
```

### Extending

- **Add metrics**: Update `MetricsSummary` and scoring logic in `run.py`
- **Add stages**: Create new stage function (e.g., `run_preview_stage`)
- **Add fixtures**: Edit `fixtures.json`
- **Update prompts**: Run `pnpm eval:export-prompts` after prompt changes

### Common Issues

**Ollama not running:**
```bash
ollama serve
```

**Model not found:**
```bash
ollama pull qwen3-vl:8b
```

**Import errors:**
```bash
uv sync --refresh
```

**Fixtures not found:**
Ensure paths in `fixtures.json` are relative to `evals/hairstyle/`.
